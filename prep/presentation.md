# Presentation Plan

## Hook
* “When you think of a turn‑based game, you usually imagine something slow, controlled, and deeply strategic. But what happens if you flip that idea on its head and make it chaotic, fast‑paced, and all about predicting your opponent? That’s exactly the kind of game I built, and that’s what I’ll be showing you today.”

## Process
* explain what `Matter.js` is and how I learned how to use it
  * 2D physics engine that handles collisions, gravity, forces, sensors, etc
* Show the code and how all the functions connect to eachother and give a quick overview of the movement and timer functions
* Images or gif of examples of using ai to help keep track of what I needed to do and help me to find what I needed to do to fix issues or make mechanics
* Show examples of large bugs I had to fix
* Show a simplified diagram or flowchart of the game loop
  * P1 Input -> Spacebar -> P2 Input -> Spacebar -> Movement -> reset Inputs & etc -> Repeat 

## Product
* Show a live demo of the game being played for a few turns
  * Make sure to show off `jumping on head` mechanic
    * could use a gif to show this mechanic instead of trying to get it to happen during the presentation
  * Show how both players input in moves before the movement phase
  * https://darrenl5941.github.io/sep11-freedom-project/

## Conclusion
* Takeaways
  * The tools you choose to use matter a lot and can change what you are able to do (matter.js vs p5js)
  * Planning and organization is important so you don't get lost in what you are doing
  * Account for time you will need to take to solve problems
  * AI can help you and help you to learn more if you know how to use it right
* Final Thoughts:
  * Looking at something that already exists and putting your own twist on it can help you to create something unique
